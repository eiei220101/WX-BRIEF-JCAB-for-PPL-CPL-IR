# WX Briefing を使う・インターネットに出す（具体手順・だれでも版）

この文書は **「何を」「どこで」「どう確認するか」** を、できるだけ細かく書いたものです。  
パソコンに不慣れでも、**上から順に**やれば進められるようにしてあります。

---

## 0. このツールでできること（先に整理）

**WX Briefing** は、設定ファイル（`config.json`）に書いたアドレスから、天気図・予報図・METAR・TAF などを取りに行き、**PDF や ZIP にまとめる**ためのプログラムです。

次の **2通り** の使い方があります。

| 番号 | 内容 | 向いている人 |
|------|------|----------------|
| **①** | **自分のパソコンだけ**で動かす | 自分の PC のブラウザで十分なとき |
| **②** | **インターネット上の URL** で開けるようにする | スマホや他の人にも同じ画面を見せたいとき |

- **①** は、Windows の **コマンドプロンプトまたは PowerShell** に短いコマンドを数回打ちます。  
- **②** は、**GitHub**（コード置き場）と **Streamlit Community Cloud**（無料の実行サービス）を使います。**サーバを自分で借りる必要はありません。**

---

## 用語（ここだけ一度読む）

| 用語 | 意味 |
|------|------|
| **ツールのフォルダ** | `app.py`・`streamlit_app.py`・`config.json` が並んでいるフォルダ。人によって場所が違います。 |
| **PowerShell / コマンドプロンプト** | Windows に付いている「文字だけの窓」。**どちらでもよい**です。ここにコマンドを打ちます。 |
| **コマンド** | 英語や記号の1行を打って **Enter** キーを押すこと。 |
| **カレントフォルダ** | 今、コマンドが「どのフォルダの中にいるか」。`cd` で変えます。 |
| **GitHub** | インターネット上にプロジェクトを置けるサービス。無料で始められます。 |
| **Streamlit Cloud** | GitHub のプログラムを動かして **URL を発行**してくれるサービス。 |

---

# ① 自分のパソコンだけで使う（Streamlit 画面）

## ①-0. 事前チェック（Python が入っているか）

1. **Windows キー** を押す。  
2. **`cmd`** と入力して Enter（**コマンドプロンプト**でも可）。  
3. 黒い窓が開いたら、次の1行を打って Enter。

   ```text
   python --version
   ```

**よい例：** `Python 3.11.x` のように **3.10 以上**の数字が出る。  
**だめな例：** `'python' は、内部コマンドまたは外部コマンド…` と出る → **Python が未インストールか PATH 未設定**です。

- 未インストールの場合：ブラウザで **<https://www.python.org/downloads/windows/>** から **Windows installer（64-bit）** をダウンロードし、インストールします。  
- インストール画面の途中に **「Add python.exe to PATH」** というチェックがあれば **必ずオン**にしてから進めます。  
- インストール後は **一度 PC を再起動するか**、開き直した **新しい** コマンドプロンプトで、もう一度 `python --version` を試します。

---

## ①-1. ツールのフォルダの「場所」を手元にメモする

次の **どちらか好きな方**で大丈夫です。

### 方法A：エクスプローラーのアドレスバーからコピーする

1. エクスプローラーを開く。  
2. `streamlit_app.py` があるフォルダまで開く。  
3. ウィンドウ **上段のアドレスバー**（フォルダ名が並んでいる所）を **一度クリック**すると、 `C:\Users\...\wx-briefing-portal` のような **1行の文字**に変わります。  
4. その文字を **すべて選択してコピー**（`Ctrl + A` → `Ctrl + C`）。  
5. メモ帳に貼り付けておく。

### 方法B：フォルダの中でコマンドプロンプトを開く（おすすめ）

1. エクスプローラーで **`streamlit_app.py` が見えているフォルダ**を開く。  
2. フォルダの **何もない白いところ**で **Shift を押しながら右クリック**。  
3. メニューに **「コマンドプロンプトをここで開く」** や **「PowerShell ウィンドウをここで開く」**、**「ターミナルで開く」** があれば、**好きな方**を選ぶ（**コマンドプロンプトで十分**です）。  
4. 開いた窓では、**もう「そのフォルダにいる」状態**なので、次の **「①-2 の cd」はスキップしてよい**です。その場合は **①-3** から進んでください。

---

## ①-2. コマンドプロンプトを開いて、フォルダに移る（方法Aを使う人だけ）

1. **Windows キー** → **`cmd`** と入力 → Enter（**コマンドプロンプト**が開きます。PowerShell でも可）。  
2. 開いた窓で、メモしたパスを使い、次のように打つ（**自分のパスに置き換える**）。

   ```powershell
   cd C:\Users\あなたの名前\...\wx-briefing-portal
   ```

   - パスに **スペース（空白）** が含まれるときは、**全体をダブルクォートで囲む**と安全です。

     ```powershell
     cd "D:\天気資料\wx-briefing-portal"
     ```

3. Enter を押す。

**うまくいったときの目安：** 窓の左端の表示が、そのフォルダのパスに変わる、またはエラーが出ない。

**よくあるエラー**

- `パスが存在しない` のような内容 → **フォルダの場所が違う**。もう一度エクスプローラーで確認する。  
- 日本語キーボードで **`¥`** が入ってしまう → パスは基本 **`\`（バックスラッシュ）** です。コピーした文字をそのまま使うのが確実です。

---

## ①-3. 必要なライブラリを入れる（初回・または更新後）

コマンドプロンプトで、**すでにツールのフォルダにいる状態**で、次を **1行ずつ** 実行します。

```powershell
python -m pip install -r requirements.txt
```

**なぜ `python -m pip` か：** 環境によって `pip` だけだと認識されないことがあるため、**確実な書き方**にしています。

**うまくいったときの目安：** たくさん英文が流れ、最後に **Successfully installed** や **Requirement already satisfied** が出る。  
**時間：** 回線次第で **1〜5分** かかることがあります。ウィンドウを閉じずに待ちます。

**だめなとき**

- `No module named pip` → Python の入れ方を見直す（①-0）。  
- `requirements.txt が見つからない` → **カレントフォルダが違う**。`dir` と打って Enter し、`requirements.txt` が一覧に出るか確認する。出なければ **①-2** からやり直す。

---

## ①-4. Streamlit 画面を起動する

同じくツールのフォルダで：

```text
python -m streamlit run streamlit_app.py
```

※ Windows では `streamlit run ...` だけだと **「認識されません」** になることがあります。そのときは **必ず上のように `python -m streamlit` から始めてください**（`python` が使えることが前提です）。

**うまくいったときの目安**

- 数秒後に **ブラウザが自動で開く**。  
- またはコマンドプロンプトに **`Local URL: http://127.0.0.1:8501`** のような行が出る → その **`http://...` から始まる1行**をコピーし、ブラウザのアドレス欄に貼り付けて Enter。

**ポートについて：** デフォルトでは **8501** 番が多いです。すでに別の Streamlit が動いていると **8502** になることもあります。**画面に出ている数字**をそのまま使ってください。

**Windows のファイアウォール**が「許可しますか？」と出たら、**プライベートなネットワークなら許可**でよいことが多いです。

---

## ①-5. ログインして画面を使う

1. ブラウザに **ログイン画面**が出ます。  
2. **Username** と **Password** を入力し、**ログイン**ボタンを押します。

**パスワードの「正解」はどこに書いてある？**

次の **上から順に** 使われます。

1. **`.streamlit/secrets.toml`** というファイルがあり、その中に `AUTH_USERNAME` と `AUTH_PASSWORD` が書いてある  
   → **それが正解**（ローカルで Secrets を作った上級者向け）  
2. 上が無い場合 → **`config.json`** の中の **`http_auth`** の **`username`** と **`password`** が正解

`config.json` の該当部分は、だいたい次のような形です（`enabled` が `true` である必要があります）。

```json
"http_auth": {
  "enabled": true,
  "realm": "WX Briefing",
  "username": "（あなたが設定したID）",
  "password": "（あなたが設定したパスワード）"
}
```

**ログイン後にできること（画面の流れ）**

- **METAR・TAF**：空港にチェック → METAR/TAF にチェック → **PDF を生成** → **ダウンロード**  
- **各種天気図・予報図**：**結合 PDF を生成** または **ZIP を生成** を押す → 準備ができたら **ダウンロード** ボタンが出る  
- **資料一覧**：折りたたみを開くと、1件ずつダウンロード用のボタンがある（初回は少し待つことがある）

---

## ①-6. 終了するとき

Streamlit を動かしている **コマンドプロンプト（または PowerShell）の窓を選んだ状態**で、キーボードの **`Ctrl` を押しながら `C`** を1回押します。  
止まったらウィンドウを閉じて構いません。

# ② インターネットの URL で使う（GitHub + Streamlit Cloud）

ここは **手順が多い**ですが、**一度できれば**あとは URL を開くだけで済みます。  
**30分〜2時間**くらい見ておくと安心です（はじめての登録作業のため）。

---

## ②-1. やることの全体像（地図）

1. **GitHub** に無料アカウントを作る（ある人はスキップ）。  
2. **新しいリポジトリ**（プロジェクトの置き場）を1つ作る。  
3. **このツールのファイル**を、そのリポジトリの **ルート**にアップロードする。  
4. **Streamlit Community Cloud** にログインし、**GitHub とつなぐ**。  
5. **New app** で、リポジトリと **`streamlit_app.py`** を指定して **Deploy**。  
6. **Secrets** にログイン用の ID・パスワードを書いて **保存**し、アプリを **再起動**。  
7. 表示された **URL** をブラウザで開く。

---

## ②-2. GitHub の準備（アカウントと空の置き場）

1. ブラウザで **<https://github.com/>** を開く。  
2. 右上 **Sign up** でアカウント作成（メール認証が来るので、メールを開いてリンクを押す）。  
3. ログイン後、右上 **「+」** → **New repository**。  
4. **Repository name** に英語の名前（例：`wx-briefing`）。  
5. **Public** でよい（無料プランで Streamlit と連携しやすい）。  
6. **Create repository** を押す。

---

## ②-3. ファイルを GitHub に載せる（画面からアップロードする方法）

「Git コマンドは使わない」前提で、**ブラウザだけ**で進めます。

1. さきほど作った **空のリポジトリのページ**を開く。  
2. **「uploading an existing file」** という青いリンク、または **Add file** → **Upload files** を押す。  
3. パソコン側の **エクスプローラー**から、次のファイルを **ドラッグ＆ドロップ**する（可能なら **全部**）。

   **必ず載せたいファイル（例）**

   - `app.py`  
   - `streamlit_app.py`  
   - `config.json`  
   - `requirements.txt`  

   **あるとよいもの**

   - `.streamlit` フォルダ（中に `secrets.toml.example` や `config.toml` が入っているならフォルダごと）  
   - `Streamlit手順.md`（この説明書）  

4. ページ下の **Commit changes**（変更を保存）を押す。  
5. リポジトリのトップに戻り、**ファイル一覧に上げた名前が並んでいる**ことを確認する。

**セキュリティで大事なこと**

- **`config.json` に平文のパスワードが入っている**状態で Public にすると、**世界中にパスワードが見える**可能性があります。  
- **おすすめ：** GitHub に上げる前に `config.json` を開き、

  - **`http_auth` の `password` を空文字 `""` にする**、または  
  - **`"enabled": false`** にする  

  そして **ログインは次の ②-6 の Secrets だけ**にします（ID・パスワードは GitHub に載せない）。

---

## ②-4. Streamlit Community Cloud に接続する

1. ブラウザで **<https://share.streamlit.io/>** を開く。  
2. **Continue with GitHub** でログイン。  
3. GitHub 側で **Streamlit にリポジトリを読ませてよいか**聞かれたら、手順に従って **許可**する。

---

## ②-5. アプリを新規作成する（New app）

1. Streamlit Cloud の画面で **「New app」** を押す。  
2. 次の項目を埋める。

   - **Repository**：②-2 で作ったリポジトリを選ぶ。  
   - **Branch**：多くの場合 **`main`**（リポジトリによっては `master` のこともある）。  
   - **Main file path**：次の **正確なファイル名**を入れる。

     ```text
     streamlit_app.py
     ```

3. **このツールがリポジトリの直下にある**なら、このまま **Deploy** でよいです。

4. **リポジトリの中に、さらにフォルダがある**場合（例：リポジトリ名 `my-work` の中に `wx-briefing-portal` がある）  
   - **Advanced settings** を開く。  
   - **Subdirectory** や **App root** に、そのフォルダ名を入れる（例：`wx-briefing-portal`）。  
   - **Main file path** は **`streamlit_app.py`** のまま（フォルダ名は Subdirectory 側に書く）。

5. **Deploy** を押して **数分待つ**。  
6. 成功すると **`https://xxxxxx.streamlit.app`** のような **URL** が表示される。**メモ帳にコピー**しておく。

**失敗したとき**

- Streamlit の画面に **ログ（Log）** を開くボタンがあることが多いです。赤いエラーの英文を **そのままコピー**しておくと、検索や質問に使えます。  
- 多い原因：**`requirements.txt` が無い**・**`streamlit` が requirements に無い**・**`config.json` が GitHub に無い**。

---

## ②-6. Secrets（ログイン情報）を Cloud にだけ登録する

1. Streamlit Cloud で **自分のアプリ**のページを開く。  
2. 右下の **「Manage app」** や **歯車（Settings）** から **Secrets** を開く。  
3. 次の **形**で書く（**引用符 `"` は英語のダブルクォート**が無難です）。

   ```toml
   AUTH_USERNAME = "FO15"
   AUTH_PASSWORD = "あなたが決めたパスワード"
   ```

4. **Save** を押す。  
5. **Reboot app**（再起動）や **Redeploy** があれば実行する。  
6. ブラウザで **②-5 でメモした URL** を開き直し、**①-5 と同じようにログイン**できるか確認する。

**ここでのパスワードは GitHub に書かない**のがポイントです。**Streamlit Cloud の Secrets だけ**に書きます。

---

## ②-7. 動作確認のチェックリスト

- [ ] GitHub のファイル一覧に **`streamlit_app.py`** と **`config.json`** と **`requirements.txt`** がある  
- [ ] Streamlit の画面が **エラーなく開く**  
- [ ] **ログイン**できる（Secrets または config のどちらかに正しい ID/パスワードがある）  
- [ ] **結合 PDF** か **ZIP** のどちらかが、エラーなく最後まで進む（設定による）

## 困ったとき（症状別）

| 症状 | 試すこと |
|------|-----------|
| `python` が認識されない | ①-0。Python を入れ直し、**PATH に追加**にチェック。新しいコマンドプロンプトで再試行。 |
| `cd` でフォルダに入れない | パスをエクスプローラーから **もう一度コピー**。空白があるなら **`cd "..."`**。 |
| `pip install` で止まる | 通信待ち。**5分以上**待つ。会社ネットワークならプロキシの制限のことも。 |
| Streamlit が開かない | コマンドプロンプトに出た **`http://127.0.0.1:数字`** を手でブラウザに貼る。 |
| `'streamlit' は認識されません` | **`python -m streamlit run streamlit_app.py`** で起動する。先に **`python -m pip install -r requirements.txt`** を実行する。 |
| ログインできない | **大文字小文字**・**全角半角**・**前後の空白**を確認。Secrets と config の **どちらが有効か**を確認。 |
| Cloud でビルド失敗 | Log を開く。**`requirements.txt`** と **`config.json`** の有無を GitHub で確認。 |

## 補足：従来の「python app.py」との違い

| 項目 | `python app.py` | `python -m streamlit run streamlit_app.py` |
|------|-----------------|-----------------------------------|
| 用途 | 自分の PC のブラウザ向け（`127.0.0.1`） | **同じ機能を Streamlit 画面で**／**Cloud で URL 公開** |
| ポート | `config.json` の `port`（例：18876） | 多くは **8501**（表示に従う） |
| 認証 | `http_auth`（Basic） | **Secrets 優先**、なければ `http_auth` |

**両方とも同じ `config.json`** を読みます。設定を変えたら **どちらを使う場合も**、動かしている側を **一度止めてから**再起動してください。
